- 亚声速：\(速度 < 声速\)，喷管 “收窄加速、扩宽减速”；
- 超声速：\(速度 > 声速\)，喷管 “扩宽加速、收窄减速”。

要理解超声速流动中 “喷管扩宽加速、收窄减速” 的规律，需突破 **“不可压缩流体” 的直觉经验 **，关注**可压缩流体（气体）在超声速下的特殊行为**，核心在于**气体的可压缩性和马赫数的物理意义**：

### 1. 先明确：声速是 “扰动传播的临界速度”

声速 c 是气体中**压力扰动（如声波）的传播速度**，反映了气体 “抵抗压缩” 的能力。当流体速度 v 超过声速（\(Ma>1\)）时，流动的 “扰动”（如压力、密度变化）**无法逆流传播**，气体的可压缩性会以和亚声速完全不同的方式体现。

### 2. 从 “质量守恒” 和 “能量守恒” 分析

对于定常流动，**质量守恒**要求 “单位时间通过任意截面的质量流量相等”，即：\(\rho v A = 常数\)（\(\rho\) 是气体密度，v 是流速，A 是喷管截面积）

同时，**能量守恒**（**等熵**流动下）要求 “气体的内能与动能相互转换”：速度增加时，气体密度会**急剧减小**（因为动能增加，内能减少，气体膨胀）。

### 3. 超声速时的特殊逻辑（\(Ma>1\)）

当 \(v > c\)（超声速），如果要让速度**继续增加**，由于动能大幅增加，气体必须**剧烈膨胀**（密度\(\rho\)急剧减小）。根据质量守恒 \(\rho v A = 常数\)，\(\rho\) 减小的幅度远大于 v 增加的幅度，因此**必须增大截面积 A**，才能保证质量流量守恒。

反之，如果喷管**收窄**（A 减小），为了满足质量守恒，要么 v 减小，要么 \(\rho\) 增大，但超声速下气体 “膨胀趋势” 极强，密度无法大幅增大，因此**速度会被迫减小**。

### 4. 举个直观例子：火箭喷管

火箭发动机的喷管是**先收窄（喉道）、后扩宽**的形状：

- 燃气在喉道前是亚声速，收窄加速到喉道处达到声速；
- 喉道后是扩张段，燃气在扩张段内**超声速流动且持续加速**，最终以极高速度喷出，产生巨大推力。

这就是 “超声速扩宽加速” 的实际应用，若没有扩张段，超声速燃气的速度会被限制，推力大幅降低。

### 总结：“反直觉” 的本质是**可压缩性的主导作用**

亚声速流动中，气体可压缩性弱，近似 “不可压缩”（比如水流），所以 “收窄加速、扩宽减速” 符合直觉；但超声速下，气体可压缩性极强，“膨胀 - 密度剧减” 的效应主导了流动规律，因此呈现 “扩宽加速、收窄减速” 的反直觉现象。



**有限差分方法**是一种 “把连续的微分方程变成离散的代数方程，再用计算机计算” 的数值工具，常用于像喷管流动这类复杂流体问题的求解。



##### 7.3节算例

**气体从大储气罐（压力温度恒定）流入 “收缩 - 扩张喷管”，最终以超声速流出**，我们要计算喷管内每个位置的密度、速度、温度、压力这些参数，最后和理论精确解对比，验证方法的正确性。

关键规律：

1. 喷管形状：先收缩（截面积变小）再扩张（截面积变大），中间最细的地方叫 “喉部”（A/A*=1）。
2. 流动特点：收缩段气体从亚声速（Ma<1）加速，喉部刚好达到声速（Ma=1），扩张段继续加速到超声速（Ma>1）。
3. 核心约束：质量守恒、动量守恒、能量守恒（等熵流动，没有能量损失）。

状态方程是描述**流体的压力、密度、温度三者之间关系**的公式，**理想气体状态方程**：p = $ \rho $ RT

p 是压力，$\rho$ 是密度，T 是温度，R 是气体常数（比如空气的 $R \approx 287 \, \text{J/(kg·K)}$）

###### **编程流程：**

步骤 1：初始化参数

先设定所有固定参数，对应书里的假设：

- 气体参数：比热比 γ=1.4（空气的标准值），Courant 数 C=0.5（稳定条件，必须≤1）；
- 喷管参数：喷管长度 L=3.0（无量纲），网格点数 N=31（x 从 0 到 3.0，Δx=0.1）；
- 喷管形状：抛物线型（书里式 7.73），核心是计算每个网格点的截面积比 A/A*（喉部 x=1.5 处 A/A*=1）；
- 初始条件：t=0 时，参数随 x 线性变化（书里式 7.74），比如 ρ'=1-0.3146x，V'=(0.1+1.09x)*sqrt (T')，T'=1-0.023x。

步骤 2：离散控制方程（核心，对应书里 7.3.1）

书里推导了 3 个控制方程（连续、动量、能量）的无量纲离散式，直接翻译成 Python 的数组运算即可，核心是 “差分替换导数”：

- 向前差分：f’(x) ≈ [f (x+Δx) - f (x)] / Δx（预测步用）；
- 向后差分：f’(x) ≈ [f (x) - f (x-Δx)] / Δx（修正步用）。

举个例子（连续方程的预测步，书里式 7.51）：

```python
# 预测步：计算密度的时间导数（向前差分）
d_rho_dt_pred = -rho[i] * (V[i+1] - V[i])/dx - rho[i]*V[i]*(np.log(A[i+1]) - np.log(A[i]))/dx - V[i]*(rho[i+1] - rho[i])/dx
```

动量方程、能量方程同理，只是公式不同，直接照书里的式 7.52、7.53 翻译，变量都是数组（每个网格点的数值）。

步骤 3：**预测步**（估算 t+Δt 的参数）

1. 对每个内点（不是边界点），用向前差分计算 ρ、V、T 的时间导数（d_rho_dt_pred、d_V_dt_pred、d_T_dt_pred）；
2. 计算时间步长 Δt：按 CFL 条件（书里式 7.67），Δt=C*Δx/(a [i]+V [i])，a 是当地声速（a=sqrt (T [i])，无量纲），取所有网格点 Δt 的最小值（保证稳定）；
3. 估算预测值：**ρ_pred = ρ + d_rho_dt_pred * Δt**，V_pred、T_pred 同理。

步骤 4：**修正步**（修正预测值）

1. 用预测值 ρ_pred、V_pred、T_pred，对每个内点用向后差分计算时间导数（d_rho_dt_corr、d_V_dt_corr、d_T_dt_corr）；
2. 取时间导数的平均值：d_rho_dt_avg = 0.5*(d_rho_dt_pred + d_rho_dt_corr)；
3. 更新最终的 t+Δt 参数：**rho_new = rho + d_rho_dt_avg * Δt**，V_new、T_new 同理。

步骤 5：处理边界条件（容易踩坑） 

书里的边界条件要严格遵守：

- 入口（i=0，亚声速）：给定 ρ'=1.0、T'=1.0（储气罐参数），速度 V1 通过内点 V2、V3 外插（V1=2*V2 - V3，书里式 7.70）；
- 出口（i=N-1，超声速）：所有参数（ρ、V、T）都通过内点外插（比如 V_out=2*V [N-2] - V [N-3]，书里式 7.72a）。

步骤 6：迭代到稳态

重复 “步骤 2 - 步骤 5”，直到相邻两次迭代的参数变化小于阈值（比如 1e-6），说明达到稳态。最后计算压力 p=ρ*T（无量纲），马赫数 Ma=V/sqrt (T)，和书里的精确解对比。







###### **无量纲化**：

（即物理量除以一个 “参考值”，初始压力 $p_0$、密度 $\rho_0$、温度 $T_0$）：让编程时的热力学参数计算更高效

- 无量纲密度 $\rho' = \rho / \rho_0$
- 无量纲温度 $T' = T / T_0$
- 无量纲压力 $p' = p / p_0$

把这些代入，得到：$p_0 p' = (\rho_0 \rho') \cdot R \cdot (T_0 T')$两边同时除以 $p_0$，整理后：$p' = \rho' \cdot \left( \frac{\rho_0 R T_0}{p_0} \right) \cdot T'$

近似为1，即：*p* ′=*ρ* ′⋅*T* ′。



###### **比热比：**

 比热比$\gamma = 1.4$（空气的标准值）

比热比$\gamma$是定压比热容$c_p$与定容比热容$c_v$ 的比值，即：$\gamma = \frac{c_p}{c_v}$

- $c_p$：单位质量的气体在**压力不变**时，温度升高 1K 所需的热量。
- $c_v$：单位质量的气体在**体积不变**时，温度升高 1K 所需的热量。

比热比反映了气体在**等熵过程**（无能量损失、可逆的绝热过程）中，温度、压力、密度等状态参数的变化规律。在气体动力学（如喷管流动、超声速流动）中，它是描述气体热力学特性的**核心参数**，直接影响：

- 声速的计算（声速公式$a = \sqrt{\gamma RT}$，其中R是气体常数，T是温度）；
- 等熵过程中压力、密度、温度的关系（如$\frac{p_2}{p_1} = \left( \frac{\rho_2}{\rho_1} \right)^\gamma = \left( \frac{T_2}{T_1} \right)^{\frac{\gamma}{\gamma-1}}$）。



###### **控制方程：**（有待理解。。。）

一、连续方程（质量守恒）

**物理意义**：“流体的质量不会凭空产生或消失”。想象水流过喷管，每个位置的 “流入质量” 必须等于 “流出质量 + 控制体内质量的变化”。

- 无量纲化后，方程描述的是：

  **「密度随时间的变化」 + 「密度与速度的空间变化」 + 「密度速度随截面积的变化」 = 0**

- 离散到编程中（预测步的向前差分）：

  用相邻网格点的

  差值 ÷ 网格步长

  （(f[i+1]-f[i])/dx）来近似 “空间变化率”，进而得到「密度的时间变化率d_rho_dt」。

  对应代码中的式子：

  ```python
  d_rho_dt = -rho[i] * (V[i+1] - V[i])/dx - rho[i]*V[i]*(np.log(A[i+1]) - np.log(A[i]))/dx - V[i]*(rho[i+1] - rho[i])/dx
  ```

  每个项的物理含义：

  - `rho[i] * (V[i+1] - V[i])/dx`：速度的空间变化导致的 “质量流走速率”；

  - `rho[i]*V[i]*(np.log(A[i+1]) - np.log(A[i]))/dx`：截面积变化导致的 “质量流量变化速率”；

  - V[i]*(rho[i+1] - rho[i])/dx：密度自身的空间变化导致的 “质量变化速率”；

    这三项的负值就是 “密度随时间的变化率”（因为质量守恒要求这些变化相互平衡）。

二、动量方程（动量守恒）

**物理意义**：“流体的动量变化由‘力’驱动”。类比开车时，油门 / 刹车（力）会改变车速（动量）；流体中，压力差、密度变化会改变流速。

- 无量纲化后，方程描述的是：

  **「速度随时间的变化」 + 「速度的空间变化」 + 「压力 / 密度的空间变化」 = 0**

- 离散到编程中（预测步的向前差分）：

  同样用 “相邻网格点的差值 ÷ 步长” 近似空间变化率，得到「速度的时间变化率d_V_dt」。

  对应书里的式 7.52，代码逻辑类似：

  ```python
  d_V_dt = -V[i]*(V[i+1]-V[i])/dx - (1/gamma)*((T[i+1]-T[i])/dx + (T[i]/rho[i])*(rho[i+1]-rho[i])/dx)
  ```

  每个项的物理含义：

  - `V[i]*(V[i+1]-V[i])/dx`：速度自身的空间变化（“惯性项”，比如高速流动时的动量传递）；

  - ```
    (1/gamma)*((T[i+1]-T[i])/dx + (T[i]/rho[i])*(rho[i+1]-rho[i])/dx)
    ```

    ：压力梯度项（压力由p=rho*T决定，所以压力变化由温度和密度的空间变化共同驱动）；

    这两项的负值就是 “速度随时间的变化率”（力的作用导致速度变化）。

三、能量方程（能量守恒）

**物理意义**：“流体的能量（温度）变化由‘机械能与内能的转化’驱动”。比如高速流动时，动能和内能会相互转化（绝热压缩 / 膨胀）。

- 无量纲化后，方程描述的是：

  **「温度随时间的变化」 + 「速度导致的温度空间变化」 + 「截面积变化导致的能量转化」 = 0**

- 离散到编程中（预测步的向前差分）：

  用 “相邻网格点的差值 ÷ 步长” 近似空间变化率，得到「温度的时间变化率d_T_dt」。

  对应书里的式 7.53，代码逻辑类似：

  ```python
  d_T_dt = -V[i]*(T[i+1]-T[i])/dx - (gamma-1)*T[i]*((V[i+1]-V[i])/dx + V[i]*(np.log(A[i+1])-np.log(A[i]))/dx)
  ```

  每个项的物理含义：

  - `V[i]*(T[i+1]-T[i])/dx`：速度导致的温度空间变化（比如流速快的地方温度如何变化）；

  - ```
    (gamma-1)*T[i]*((V[i+1]-V[i])/dx + V[i]*(np.log(A[i+1])-np.log(A[i]))/dx)
    ```

    ：动能与流动功转化为内能的部分（

    gamma-1是热力学系数，反映内能和动能的转化关系）；

    这两项的负值就是 “温度随时间的变化率”（能量转化导致温度变化）。

总结：三个方程的 “编程逻辑共性”

这三个方程都是 **“守恒定律” 的数学表达 **，离散时都用 **“相邻网格点的差值 ÷ 步长” 来近似空间变化率 **，进而得到 “时间变化率”，最终在预测 - 修正步中更新每个网格点的密度、速度、温度。

你可以把这三个方程想象成 “三个联动的计算器”：

- 连续方程算「密度怎么变」；

- 动量方程算「速度怎么变」；

- 能量方程算「温度怎么变」；

  它们一起迭代，直到喷管内的流场达到稳定（稳态）。

#include <iostream>
#include <cuda_runtime.h>

// CUDA 错误检查宏
#define CUDA_CHECK(err) { \
    cudaError_t error = err; \
    if (error != cudaSuccess) { \
        fprintf(stderr, "CUDA Error at %s:%d - %s\n", __FILE__, __LINE__, cudaGetErrorString(error)); \
        exit(EXIT_FAILURE); \
    } \

// 1. 核函数定义
// 在 GPU 上执行向量加法
__global__ void vectorAddKernel(const float *A, const float *B, float *C, int N) {
    // 2. 线程组织：计算全局线程ID
    int tid = blockIdx.x * blockDim.x + threadIdx.x;

    // 边界检查，防止数组越界
    if (tid < N) {
        C[tid] = A[tid] + B[tid];
    }
}

int main() {
    int N = 1 << 20; // 1048576 个元素
    size_t size = N * sizeof(float);

    // 在主机上分配内存
    float *h_A = new float[N];
    float *h_B = new float[N];
    float *h_C = new float[N];

    // 初始化主机数据
    for (int i = 0; i < N; ++i) {
        h_A[i] = static_cast<float>(rand()) / RAND_MAX;
        h_B[i] = static_cast<float>(rand()) / RAND_MAX;
    }

    // 3. & 4. GPU 内存管理与数据传输
    float *d_A, *d_B, *d_C;

    // 在设备上分配全局内存
    CUDA_CHECK(cudaMalloc(&d_A, size));
    CUDA_CHECK(cudaMalloc(&d_B, size));
    CUDA_CHECK(cudaMalloc(&d_C, size));

    // 将数据从主机拷贝到设备
    CUDA_CHECK(cudaMemcpy(d_A, h_A, size, cudaMemcpyHostToDevice));
    CUDA_CHECK(cudaMemcpy(d_B, h_B, size, cudaMemcpyHostToDevice));

    // 配置核函数启动参数
    int threadsPerBlock = 256;
    // 计算需要的线程块数量，确保能覆盖所有N个元素
    int blocksPerGrid = (N + threadsPerBlock - 1) / threadsPerBlock;

    // 启动核函数
    vectorAddKernel<<<blocksPerGrid, threadsPerBlock>>>(d_A, d_B, d_C, N);
    
    // 检查核函数启动是否有错误
    CUDA_CHECK(cudaGetLastError());
    CUDA_CHECK(cudaDeviceSynchronize()); // 等待GPU完成所有任务

    // 将结果从设备拷贝回主机
    CUDA_CHECK(cudaMemcpy(h_C, d_C, size, cudaMemcpyDeviceToHost));

    // 验证结果
    bool success = true;
    for (int i = 0; i < 10; ++i) { // 只验证前10个
        if (abs(h_C[i] - (h_A[i] + h_B[i])) > 1e-5) {
            success = false;
            break;
        }
    }
    if (success) {
        std::cout << "Vector addition successful!" << std::endl;
        std::cout << "A[0] = " << h_A[0] << ", B[0] = " << h_B[0] << ", C[0] = " << h_C[0] << std::endl;
    } else {
        std::cout << "Vector addition failed!" << std::endl;
    }

    // 释放内存
    delete[] h_A;
    delete[] h_B;
    delete[] h_C;
    CUDA_CHECK(cudaFree(d_A));
    CUDA_CHECK(cudaFree(d_B));
    CUDA_CHECK(cudaFree(d_C));

    return 0;
}
```

###### 隐式方法：Crank-Nicolson 格式是什么？（为什么要⽤隐式？）

文档用**Crank-Nicolson 隐式有限差分方法**求解式 (8.20)，核心是解决 “显式方法的时间步限制”—— 显式方法受 CFL 准则约束，Delta t必须很小，否则不稳定；而隐式方法**无条件稳定**，可以用大Delta t，大幅减少计算步数。

“局部贡献” 指的是单个网格单元对全局方程组（\**KU=F\**）的单元级贡献，也就是局部刚度矩阵 `local_K` 和局部载荷向量 `local_F`。

悬点与普通节点的积分项是 “独立可分” 的

在单元的积分计算中，悬点和普通节点对应的积分项是**相互独立**的 —— 悬点的积分项只依赖其插值后的形函数和自身的积分域子区域，普通节点的积分项只依赖自身的形函数和积分域子区域，二者之间没有 “计算依赖”：

- 普通节点的积分项：∫Ω∇N普通⋅∇N普通dΩ
- 悬点的积分项：∫Ω∇N悬点（插值）⋅∇N普通dΩ

这两个积分项是 “并行的数学运算”，先算哪一个，都不会改变各自的积分结果，最终合并后的局部贡献值也完全相同。
