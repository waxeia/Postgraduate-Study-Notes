1. **隐式格式核心概念**：回顾第 4 章 Crank-Nicolson 隐式方法的基本思想（时间上取 t 和 t+Δt 的平均，无条件稳定），对比第 7 章显式方法的差异（显式受 CFL 准则限制 Δt，隐式无此约束）。

### 二、物理问题与精确解拆解（先懂 “算什么”，再学 “怎么算”）

在动手离散和编码前，必须吃透 Couette 流动的物理本质（文档 8.2 节核心内容），这是数值解验证的依据：

1. 物理模型理解

   ：

   - 流动场景：两平行平板间距为 D，下板静止（u=0），上板以速度 uₑ运动，流体为不可压缩黏性流体，无压力梯度（∂p/∂x=∂p/∂y=0）。
   - 核心特性：流场仅沿 y 方向变化（u=u (y)），v=0（无垂直方向速度），稳态速度剖面为**线性分布**（精确解：u/uₑ = y/D）。

2. 控制方程简化

   ：

   - 从纳维 - 斯托克斯方程出发，结合 Couette 流动的假设（∂/∂x=0、v=0、定常→非定常保留时间项），最终简化为无量纲化的抛物型方程：

     

     ∂u/∂t = (1/Re)·∂²u/∂y²

     

     （u、t、y 为无量纲量，Re 为雷诺数，Re=ρuₑD/μ）。

3. **精确解牢记**：稳态时 u (y) = (uₑ/D) y，这是后续数值解收敛的 “标准答案”，需时刻用于对比验证。

### 三、核心方法：控制方程离散与三对角方程组构建（关键步骤）

这是隐式计算的核心，需严格遵循文档 8.3.1 节的离散逻辑：

1. 网格划分

   ：

   - 设定 y 方向网格点数量（文档用 21 个点，N+1=21），则网格间距 Δy = 1/20（无量纲化后 y∈[0,1]）。
   - 明确未知量：网格点 j=2~20（j=1 为下板 u=0，j=21 为上板 u=1，边界条件已知）。

2. Crank-Nicolson 格式离散

   ：

   - 对无量纲控制方程进行离散，利用 “t 时刻和 t+1 时刻的二阶导数平均”，整理得到三对角方程组形式：

     

     A·uⱼ₋₁ⁿ⁺¹ + B·uⱼⁿ⁺¹ + A·uⱼ₊₁ⁿ⁺¹ = Kⱼ

   - 牢记系数推导结果（文档 8.37a~c）：

     

     A = -E/2，B = 1+E，Kⱼ = (1-E)uⱼⁿ + (E/2)(uⱼ₊₁ⁿ + uⱼ₋₁ⁿ)

     

     （E 为无量纲时间步参数，E=Δt/(Re・(Δy)²)，决定 Δt 大小）。

3. 边界条件处理

   ：

   - 下板（j=1）：u₁=0，代入方程组后首行变为 B・u₂ⁿ⁺¹ + A・u₃ⁿ⁺¹ = K₂；
   - 上板（j=21）：u₂₁=1，代入方程组后末行变为 A・u₁₉ⁿ⁺¹ + B・u₂₀ⁿ⁺¹ = K₂₀ - A・1。

### 四、算法实现：用 Python 编写 Thomas 算法（落地执行）

这一步需将数学模型转化为代码，核心是实现 Thomas 算法求解三对角方程组：

1. 代码结构拆解

   ：

   - 第一步：定义基础参数（Re=5000、E=10（最优值）、网格点数量 N+1=21、迭代终止条件（相邻时间步速度变化 < 1e-6））；
   - 第二步：初始化速度场（t=0 时，u₁~u₂₀=0，u₂₁=1，符合文档初始条件）；
   - 第三步：实现 Thomas 算法（前向消元：简化三对角矩阵为二对角矩阵；后向回代：从末行开始求解未知量 uⱼⁿ⁺¹）；
   - 第四步：时间推进迭代（循环计算每个时间步的 uⁿ⁺¹，直到速度剖面稳定为线性）。

2. 关键代码注意事项

   - 处理三对角矩阵的系数数组（a=A，b=B，c=A，右侧向量 d=Kⱼ）；

   - 后向回代从 j=N（u₂₀）开始，反向计算到 j=2（u₂）。

##### 第一个算例：

你需要实现的第一个算例是 **基于 Crank-Nicolson 隐式格式 + Thomas 算法的不可压缩非定常 Couette 流动数值求解**（对应文档 8.3 节核心内容），也是入门隐式计算的核心基础算例，具体细节如下：

### 一、算例核心定义（明确 “做什么”）

1. **物理场景**：两平行平板间的不可压缩黏性流动，平板间距为 D（无量纲化后 y∈[0,1]），下板静止（u=0），上板以速度 uₑ运动（无量纲化后 u=1），无压力梯度，流动仅沿 y 方向变化（v=0）。

2. 控制方程

   ：无量纲化的抛物型非定常动量方程

   

   ∂u/∂t = (1/Re)·∂²u/∂y²

   

   （Re=5000，为以平板间距 D 为特征长度的雷诺数）。

3. 数值方法

   ：

   - 离散格式：Crank-Nicolson 隐式有限差分格式（无条件稳定）。
   - 求解算法：Thomas 算法（追赶法），高效求解离散后的三对角方程组。

4. **核心目标**：通过时间推进迭代，让初始非物理速度分布（t=0 时，y∈[0,1) 时 u=0，y=1 时 u=1）逐渐收敛到 Couette 流动的精确解（u=y，线性分布），验证隐式方法的稳定性和有效性。

### 二、算例关键参数与约束（明确 “用什么条件做”）

1. **网格设定**：y 方向共 21 个网格点（N+1=21），网格间距 Δy=1/20=0.05（无量纲化后），未知速度点为 j=2~20（共 19 个未知量）。

2. **初始条件**：t=0 时，u₁~u₂₀=0（下板到上板下方所有网格点），u₂₁=1（上板边界）。

3. **边界条件**：始终保持 u₁=0（下板）、u₂₁=1（上板），迭代过程中不变。

4. 关键参数

   ：

   - 雷诺数 Re=5000（文档示例取值，保证流动特性符合假设）。
   - 时间步参数 E=10（文档推荐最优值，兼顾迭代效率和瞬时精度，Δt=E・Re・(Δy)²=12.5）。
   - 收敛判据：相邻时间步的速度变化最大值 < 1e-6，或迭代至速度剖面与精确解（u=y）的误差 < 1%。

### 三、算例实现核心步骤（明确 “怎么做”）

1. 定义基础参数：Re=5000、E=10、网格点数量 = 21、Δy=0.05、迭代终止条件。
2. 初始化速度场：按初始条件生成 u 数组（u=[0,0,...,0,1]，共 21 个元素）。
3. 离散控制方程：用 Crank-Nicolson 格式对无量纲方程离散，得到三对角方程组 `A·uⱼ₋₁ⁿ⁺¹ + B·uⱼⁿ⁺¹ + A·uⱼ₊₁ⁿ⁺¹ = Kⱼ`（其中 A=-E/2=-5，B=1+E=11，Kⱼ由 t 时刻速度计算）。
4. 处理边界条件：将首行（j=2）和末行（j=20）方程简化（代入 u₁=0 和 u₂₁=1），得到简化后的三对角矩阵。
5. 实现 Thomas 算法：通过 “前向消元” 将三对角矩阵转化为二对角矩阵，再通过 “后向回代” 求解 t+1 时刻的速度场 uⁿ⁺¹。
6. 时间推进迭代：重复步骤 3~5，直到速度剖面收敛为线性分布（与精确解 u=y 吻合）。

### 四、算例验证标准（明确 “做对的标志”）

1. 收敛后速度剖面：所有网格点的 u 值满足 u=y（无量纲），与文档表 8.2 中 240 时间步后的结果误差 < 1%（例如 j=11 时 y=0.5，u≈0.5；j=20 时 y=0.95，u≈0.95）。
2. 迭代效率：E=10 时，约 36 个时间步即可收敛（远少于显式方法的迭代步数），体现隐式方法 “大 Δt、高效率” 的优势。
3. 稳定性验证：即使调整 E=40（Δt 增大），仍能稳定收敛到精确解（无振荡或发散）。

###### 步骤 10：内存访问优化（GPU 性能的核心）

- 优化动作：
  1. 数据布局优化：将 “结构体数组（AoS）” 改为 “数组结构体（SoA）”，例如：
     - 原 CPU 格式（AoS）：`struct { double rho, u, v, w, p; } q[ncells];`（GPU 访问不连续）；
     - 优化后（SoA）：`double rho[ncells], u[ncells], v[ncells], w[ncells], p[ncells];`（GPU 合并访问，效率提升 5-10 倍）；
  2. 共享内存利用：将频繁访问的数据（如相邻单元的通量、变量值）预加载到共享内存，减少全局内存访问（全局内存访问延迟是共享内存的 100 倍以上）；
  3. 减少数据拷贝：采用 “零拷贝内存”（`cudaHostAlloc`）或 “统一内存”（Unified Memory），避免 CPU-GPU 之间的频繁数据传输（尤其适合大网格算例）。

###### 步骤 11：并行效率优化

- 优化动作：
  1. 线程块大小优化：根据 GPU 架构（如 Ampere 架构的 SM 单元），设置合理的线程块大小（通常为 128、256、512，需通过 profiler 测试最优值）；
  2. 循环展开：对核函数中的内层循环（如通量导数计算的循环）进行手动展开（`#pragma unroll`），减少指令开销；
  3. 分支优化：避免线程块内的分支分歧（如`if-else`），若必须分支，尽量让同一线程块的线程走相同分支。

- 实操动作：
  1. 使用 NVIDIA Nsight Compute（GPU 性能分析工具）：
     - 查看核函数的 “内存访问效率”（目标 > 80%）、“指令吞吐量”、“分支分歧率”；
     - 定位性能瓶颈（如某核函数的全局内存访问效率低，或线程块大小不合理）；
  2. 对比优化前后的性能：记录 “单时间步耗时”“GPU 加速比”（CPU 耗时 / GPU 耗时），目标加速比≥10 倍（中等网格）、≥50 倍（大网格）。

###### 第五阶段：工程化与扩展（1 个月）—— 让代码 “可用、可维护”

目标：解决移植后的工程化问题，支持更复杂的算例和 GPU 架构。

###### 步骤 13：代码工程化优化

- 动作：
  1. 封装 GPU 接口：将 GPU 相关代码封装为独立的函数（如`gpu_time_advance()`），与 CPU 代码分离，便于后续维护和切换；
  2. 错误处理：添加 CUDA 错误检查（如`cudaGetLastError()`），避免因 GPU 内存不足、核函数启动失败等问题导致崩溃；
  3. 兼容性适配：支持不同 GPU 架构（通过`-gencode`编译选项），若需支持 AMD GPU，可基于 HIP 重构（HIP 语法与 CUDA 兼容，移植成本低）。

###### 步骤 14：扩展测试与稳定性验证

- 动作：
  1. 运行大规模算例：测试 GPU 对大网格（如 100 万 + 单元）的支持能力，验证内存是否充足、计算是否稳定；
  2. 长期运行测试：连续运行 1000 + 时间步，检查是否出现数值漂移、内存泄漏（通过`cudaMemGetInfo`监控显存使用）；
  3. 与 CPU 版本的长期结果对比：确保非稳态计算的长期演化（如湍流模拟）与 CPU 结果一致。

###### 关键注意事项（避坑指南）

1. 数值一致性优先于性能：移植初期不要追求速度，先保证结果与 CPU 一致，否则优化毫无意义；
2. 避免 “重写” 复杂算法：对于块三对角求逆、多重网格等复杂模块，优先使用成熟的 GPU 库（如 cuSolver、AmgX），手动实现容易出错且效率低；
3. 关注 GPU 显存限制：大网格算例的变量数组（如`q`、`R`、`F`）可能占用大量显存，需提前计算显存需求（如 1000 万单元的`double`型`rho`数组占用 80MB），避免显存溢出；
4. 文档同步更新：记录移植过程中的关键决策（如数据布局选择、库函数调用）、参数调整（如线程块大小），方便后续维护和迭代。
